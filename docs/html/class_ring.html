Detailed Description

Wrapper around io_uring that manages submission and completion queues. Provides methods for submitting operations and retrieving completions, abstracting low-level io_uring syscalls and memory management.

unsigned long id ) const

Submit a timeout operation to the submission queue.

This method schedules a timeout with the specified duration and flags. The timeout will trigger after count occurrences of the specified time period given by spec (typically 1 for a single timeout).

Parameters
specPointer to a __kernel_timespec struct specifying the timeout duration.
countNumber of timeout expirations before triggering (usually 1).
flagsTimeout flags controlling behavior (e.g., IORING_TIMEOUT_ABS, IORING_TIMEOUT_UPDATE, IORING_TIMEOUT_CLOCK_MONOTONIC).
idUser-defined identifier for this timeout operation, stored in the submission queue entry's user_data field.
Note
The caller is responsible for ensuring that spec remains valid until the timeout expires or is removed.

◆ submit_timeout_remove()

unsigned long id ) const

Submit a timeout removal operation to cancel a previously submitted timeout.

This operation attempts to cancel the timeout identified by the given id. The kernel uses the id to match the timeout to remove, which corresponds to the user_data value originally supplied during the submit_timeout call.

Parameters
flagsFlags controlling the removal operation (typically 0).
idThe user-defined identifier of the timeout to remove.
Note
Successful removal generates a completion event with result code -ECANCELED.

void Ring::submit_timeout_update ( __kernel_timespec * next,
unsigned int flags,
unsigned long id ) const

Submit a timeout update operation to modify an existing timeout.

This updates the expiration time of an existing timeout identified by id to the new time specified by next. The update uses the IORING_TIMEOUT_UPDATE flag.

Parameters
nextPointer to a __kernel_timespec specifying the new timeout duration.
flagsFlags controlling the update behavior.
idThe user-defined identifier of the timeout to update (matches the original timeout's user_data).
Note
The kernel expects the update to be encoded as a timeout removal operation with the IORING_TIMEOUT_UPDATE flag set and off field set to next. The caller must ensure the next pointer remains valid until the update is processed.
